# Generated Character Assets

Each character lives in its own subdirectory. Runtime keeps the procedural rig, and reads optional face and outfit skins from the `skins/` subdirectory.

Shared fallback skins live in `_shared_skins/`. If a character-specific face or outfit skin is missing, runtime falls back to the matching shared asset there.

Supported defaults generated by this script include `skins/face_*.png`, `skins/face_talk_*_open.png`, `skins/face_talk_*_closed.png`, `skins/outfit*.png`, and optional limb skins:

- `skins/upper_arm.png`
- `skins/lower_arm.png`
- `skins/upper_leg.png`
- `skins/lower_leg.png`

When limb skins are present, runtime maps them onto the corresponding arm and leg segments and rotates each segment with the underlying joints.

You can replace these with your own PNG files.
